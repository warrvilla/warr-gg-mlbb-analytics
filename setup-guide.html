Go-Live Setup Guide
Everything you need to deploy Warr.GG with invite-only access, user profiles, and live hosting. Two services — Supabase (auth) and Netlify (hosting) — both free.
1
Supabase — Auth & Database
Free · Takes about 10 minutes · supabase.com
1
Create a Supabase Project
  1. Go to supabase.com → Sign up for a free account
  2. Click New Project → Give it a name like warr-gg → Set a database password → Click Create
  3. Wait about 2 minutes for the project to be ready
2
Get Your Project URL & Publishable Key — paste into warr-lib.js

You need two values from your Supabase dashboard. Here's exactly where to find each one:

🔗
Project URL
Go to Settings → General. You'll see a Project ID field — something like nqqlmzdyyhbyvsbybdem.
Your URL is: https://YOUR_PROJECT_ID.supabase.co
Just swap in your actual Project ID.
🔑
Publishable Key (this is the anon key)
Go to Settings → API Keys. You'll see a section called Publishable key with a value starting with sb_publishable_...
Click the copy icon next to it. That is your anon key — Supabase just renamed it.
  1. Open warr-engine/warr-lib.js in any text editor
  2. Find the WARR_CONFIG block near the top (around line 9)
  3. Replace both placeholder values with your real ones:
warr-lib.js — lines 9–12
const WARR_CONFIG = {
  supabaseUrl:     'https://YOUR_PROJECT_ID.supabase.co',  // ← from Settings → General → Project ID
  supabaseAnonKey: 'sb_publishable_...',                   // ← from Settings → API Keys → Publishable key
};
⚠️
Never use the Secret key here In Settings → API Keys there's also a Secret key starting with sb_secret_... — do NOT put that in warr-lib.js. Only the Publishable key goes in the front-end code.
3
Create the Profiles Table — run this SQL once
  1. In your Supabase project, look at the left sidebar icons — click the one that looks like a terminal/code icon labeled SQL Editor
  2. Click + New query (top left of the editor)
  3. Paste the entire block below into the editor
  4. Click the green Run button (or press Ctrl+Enter)
  5. You should see Success. No rows returned at the bottom
SQL — Run once in Supabase SQL Editor
-- User profiles (your name is required, team name is optional)
CREATE TABLE public.profiles (
  id            uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  display_name  text NOT NULL,
  team_name     text,
  created_at    timestamptz DEFAULT now()
);

-- Lock it down: users can only see and edit their own profile
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view their own profile"
  ON public.profiles FOR SELECT
  USING (auth.uid() = id);

CREATE POLICY "Users can insert their own profile"
  ON public.profiles FOR INSERT
  WITH CHECK (auth.uid() = id);

CREATE POLICY "Users can update their own profile"
  ON public.profiles FOR UPDATE
  USING (auth.uid() = id);
3b
Create Scout Matches & Draft Saves Tables — run this SQL too

Still in the SQL Editor — click + New query again to open a fresh tab, then paste and run this second block. It creates the two data tables that power shared match scouting and per-user draft saving.

SQL — Run once in Supabase SQL Editor
-- ═══════════════════════════════════════════
-- SCOUT MATCHES — shared across all users
-- Anyone on the team can enter/see MPL data
-- ═══════════════════════════════════════════
CREATE TABLE public.scout_matches (
  id          text PRIMARY KEY,
  created_at  timestamptz DEFAULT now(),
  updated_at  timestamptz DEFAULT now(),
  created_by  uuid REFERENCES auth.users(id),
  data        jsonb NOT NULL
);

ALTER TABLE public.scout_matches ENABLE ROW LEVEL SECURITY;

CREATE POLICY "All users read matches"
  ON public.scout_matches FOR SELECT
  USING (auth.role() = 'authenticated');

CREATE POLICY "All users insert matches"
  ON public.scout_matches FOR INSERT
  WITH CHECK (auth.role() = 'authenticated');

CREATE POLICY "Creator updates match"
  ON public.scout_matches FOR UPDATE
  USING (auth.uid() = created_by OR created_by IS NULL);

CREATE POLICY "Creator deletes match"
  ON public.scout_matches FOR DELETE
  USING (auth.uid() = created_by OR created_by IS NULL);


-- ═══════════════════════════════════════════
-- DRAFT SAVES — private per user account
-- Each analyst's own saved Draft Board states
-- ═══════════════════════════════════════════
CREATE TABLE public.draft_saves (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     uuid REFERENCES auth.users(id) NOT NULL,
  created_at  timestamptz DEFAULT now(),
  name        text NOT NULL DEFAULT 'Draft',
  data        jsonb NOT NULL
);

ALTER TABLE public.draft_saves ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Own drafts only"
  ON public.draft_saves FOR ALL
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);
📋
What these tables do: scout_matches — Scout entries (MPL, tournaments) are shared; every user on your team sees new matches immediately. draft_saves — each person's saved Draft Board states are private to their own account. AI Battle stays fully local (no cloud sync).
4
Turn Off Public Sign-Ups (invite-only)
  1. In the left sidebar, click the person icon — Authentication
  2. Click Providers (or Sign In / Up depending on your Supabase version)
  3. Find Email and expand it — look for Enable Email Signup (or Allow new users to sign up) and turn it OFF
  4. Click Save — now only users you personally invite can create accounts
✅
You're done with Supabase for now You'll come back to set the site URL after deploying to Netlify (Step 3.2 below).
2
Netlify — Deploy the Site
Free · Takes 5 minutes · netlify.com
1
Make sure warr-lib.js has your keys filled in

Before uploading, double-check that warr-lib.js has real values — not YOUR_PROJECT_ID placeholders. If you skipped Step 1.2, go back and do that first.

2
Drag and Drop to Deploy
  1. Go to netlify.com → Sign up for a free account
  2. On the dashboard you'll see a big dropzone: "Drag and drop your site folder here"
  3. Drag your entire warr-engine/ folder onto that dropzone
  4. Netlify deploys it instantly and gives you a URL like https://cool-name-abc123.netlify.app
  5. Copy that URL — you need it for the next step
💡
Want to update the site later? Drag and drop the folder again, or connect a GitHub repo (Site Settings → Build & Deploy → Link to Git) for automatic redeploys when you push changes.
3
Tell Supabase your live URL (prevents redirect errors)
  1. In Supabase, click Authentication in the left sidebar
  2. Click URL Configuration
  3. Set Site URL to your Netlify URL:
    https://cool-name-abc123.netlify.app
  4. Under Redirect URLs, click Add URL and add both:
    https://cool-name-abc123.netlify.app
    https://cool-name-abc123.netlify.app/auth.html
  5. Click Save
🎉
Your site is live! Visit your Netlify URL. You should be redirected to the sign-in page immediately. No one can get in without your invite.
3
Inviting & Managing Users
All done from the Supabase dashboard — no coding needed
▶
Give someone access
  1. Go to Authentication → Users → click Invite User
  2. Enter their email address → click Send Invite
  3. They receive an email — when they click the link, they land on your site's sign-in page
  4. They fill in their name (required) and team name (optional), then set a password
  5. After that they can sign in any time with email + password
⏱️
Invite links expire after 24 hours If someone doesn't click in time, just invite them again from the same Users screen.
▶
Revoke access
  1. Go to Authentication → Users
  2. Find the person → click the three-dot menu → Delete User
  3. They are immediately locked out — their next request to any page redirects to sign-in
▶
Reset someone's password
  1. Go to Authentication → Users
  2. Find the user → three-dot menu → Send Password Recovery
  3. They receive an email to reset their password

Troubleshooting
Common issues and how to fix them
"Invalid login credentials"
Wrong email or password. Go to Supabase → Authentication → Users → find the user → Send Password Recovery so they can reset it.
Invite link says "expired" or doesn't work
Supabase invite links expire after 24 hours. Go to Authentication → Users → find the user (or delete them and re-invite) → Invite User again with the same email.
Site keeps redirecting to auth.html even when logged in
Usually means the Supabase URL or Publishable key in warr-lib.js is wrong. Open the file, check WARR_CONFIG at the top — the URL should be https://YOUR_PROJECT_ID.supabase.co (from Settings → General) and the key should be the Publishable key starting with sb_publishable_ (from Settings → API Keys). No extra spaces.
User gets to auth.html but "Complete Profile" form doesn't show
The Supabase redirect URL isn't set correctly. Go to Supabase → Authentication → URL Configuration and make sure both your Netlify URL and your-site.netlify.app/auth.html are listed under Redirect URLs.
Blank page or "Failed to fetch" errors
The Supabase project might be paused (free projects pause after 1 week of inactivity). Go to supabase.com, open your project, and click Restore project. Takes about 30 seconds.
Deployed site but pages show 404
Make sure you dragged the warr-engine/ folder itself, not a parent folder containing it. The file index.html must be at the root of the uploaded folder.
